# 要件定義関連ドキュメント

このフォルダには、タスク管理アプリの要件定義・設計に関するドキュメントが含まれています。

## ファイル構成

### 要件定義
- `要件定義書_タスク管理アプリ.md` - プロジェクトの要件定義書

### 設計書
- `database-design/` - データベース設計書
  - ER図、テーブル定義、SQLスクリプト
- `api-design/` - API設計書
  - RESTful API仕様、OpenAPI定義
- `layout-design/` - 画面設計書
  - HTMLモックアップ、ワイヤーフレーム

## 設計フロー

1. **要件定義** → 機能要件・非機能要件の整理
2. **画面設計** → UI/UXの設計とモックアップ作成
3. **データベース設計** → データモデルとスキーマ設計
4. **API設計** → エンドポイントとデータ形式の設計

## 参照順序

実装時は以下の順序で参照することを推奨：

1. 要件定義書で全体像を把握
2. 画面設計でUI仕様を確認
3. データベース設計でデータ構造を理解
4. API設計でインターフェースを確認
